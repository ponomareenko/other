Да, GUID в 1С можно выгрузить, но это зависит от того, **какая у вас конфигурация** (УТ, ERP, КА, Бухгалтерия и т.д.). В большинстве современных конфигураций GUID есть у каждого объекта, но **по умолчанию в списках и стандартных выгрузках он не отображается**.

## Вариант 1. Через внешнюю обработку или запрос (самый правильный)

Если есть программист 1С, то можно сделать запрос:

```1C
ВЫБРАТЬ
    Номенклатура.Ссылка КАК Ссылка,
    Номенклатура.Код КАК Код,
    Номенклатура.Артикул КАК Артикул,
    Номенклатура.Наименование КАК Наименование,
    XMLСтрока(Номенклатура.Ссылка) КАК GUID
ИЗ
    Справочник.Номенклатура КАК Номенклатура
```

или

```1C
УникальныйИдентификатор = XMLСтрока(Номенклатура.Ссылка);
```

В результате получится строка вида:

```
c9d2d91d-7ef6-11ef-80e2-0050569d1234
```

Именно её и стоит хранить в WordPress.

---

## Вариант 2. Через Консоль запросов

Если установлена «Консоль запросов»:

```sql
ВЫБРАТЬ
    Ссылка,
    Код,
    Артикул,
    Наименование,
    XMLСтрока(Ссылка) КАК GUID
ИЗ
    Справочник.Номенклатура
```

После этого результат можно выгрузить в Excel или CSV.

---

## Вариант 3. Через стандартную выгрузку

В большинстве конфигураций стандартная выгрузка в Excel:

```
Файл → Вывести список → Excel
```

**не содержит GUID**.

Она выгружает только:

* Код
* Артикул
* Наименование
* Единицу
* Цены
* и т.п.

GUID туда не попадает.

---

## Вариант 4. Через обработку обмена

Если используется обмен через XML или CommerceML, то GUID уже присутствует.

Например:

```xml
<Ид>9bfbef76-0c6c-11ef-80d1-005056010001</Ид>
```

Этот `<Ид>` — и есть GUID объекта.

# Что я рекомендую для вашего проекта

Если вы будете писать собственную синхронизацию между 1С и WooCommerce, то оптимальная схема такая:

| 1С                         | WordPress             |
| -------------------------- | --------------------- |
| GUID (`XMLСтрока(Ссылка)`) | `_1c_guid` (метаполе) |
| Артикул                    | `SKU`                 |
| Наименование               | Название товара       |
| Цена                       | Regular Price         |
| Остаток                    | Stock                 |
| Изображения                | Галерея товара        |
| Категория                  | Категория WooCommerce |

### Сам GUID в WooCommerce лучше хранить не в отдельной колонке таблицы, а в `post_meta`.

Например:

```text
meta_key   = _1c_guid
meta_value = c9d2d91d-7ef6-11ef-80e2-0050569d1234
```

Это полностью соответствует архитектуре WordPress, не требует изменения структуры базы данных и позволяет быстро находить товар по GUID при синхронизации. Если синхронизация будет выполняться регулярно, имеет смысл добавить индекс по `meta_key` и `meta_value` или реализовать отдельную таблицу соответствий для ускорения поиска на очень больших каталогах (десятки и сотни тысяч товаров). Для большинства интернет-магазинов достаточно `post_meta` с полем `_1c_guid`.
