Да, это можно сделать, и это, на мой взгляд, самый правильный способ для последующей синхронизации с WordPress.

## Способ 1. Через Консоль запросов (рекомендуется)

Если у вас есть доступ к **Консоли запросов** в 1С, то можно выполнить такой запрос:

```1C
ВЫБРАТЬ
    Номенклатура.Ссылка КАК GUID,
    XMLСтрока(Номенклатура.Ссылка) КАК GUIDСтрока,
    Номенклатура.Наименование КАК Наименование,
    Номенклатура.Артикул КАК Артикул,
    Номенклатура.ЕдиницаХраненияОстатков.Наименование КАК ЕдиницаИзмерения
ИЗ
    Справочник.Номенклатура КАК Номенклатура
ГДЕ
    НЕ Номенклатура.ПометкаУдаления
```

После выгрузки получится примерно такая таблица:

| GUID           | Наименование            | Артикул | Единица измерения |
| -------------- | ----------------------- | ------- | ----------------- |
| `6f5e7c9d-...` | Раковина антивандальная | RV-500  | шт                |
| `a1b2c3d4-...` | Писсуар настенный       | PS-100  | шт                |
| `b9d7e4f1-...` | Перегородка             | MP-200  | шт                |

---

## Способ 2. Через список номенклатуры (без программирования)

Если нет доступа к программированию:

1. Открыть **Справочники → Номенклатура**.
2. Нажать **Ещё → Изменить форму**.
3. Посмотреть, есть ли возможность добавить поле **Уникальный идентификатор** или **GUID**.

Но в большинстве типовых конфигураций такого поля **нет**, поэтому GUID этим способом получить нельзя.

---

## Способ 3. Через внешнюю обработку

Если есть программист 1С, можно сделать обработку, которая сразу формирует CSV:

```csv
GUID;Наименование;Артикул;ЕдиницаИзмерения
6f5e7c9d-...;Раковина;RV500;шт
a4b7d321-...;Писсуар;PS100;шт
```

Именно такой формат будет максимально удобен для импорта в WordPress.

---

# Важно: поле GUID

В запросе есть два варианта:

```1C
Номенклатура.Ссылка
```

и

```1C
XMLСтрока(Номенклатура.Ссылка)
```

Использовать нужно **именно `XMLСтрока(Номенклатура.Ссылка)`**.

Почему?

* `Ссылка` — это внутренний объект 1С, который неудобно использовать вне системы.
* `XMLСтрока(Ссылка)` возвращает строковый GUID вида:

```
6f5e7c9d-7ef6-11ef-80e2-0050569d1234
```

Её можно без проблем хранить в WooCommerce в метаполе `_1c_guid`, передавать через API и использовать для поиска.

## Если планируется двусторонняя синхронизация

Я бы рекомендовал выгружать сразу такой минимальный набор полей:

| Поле                       | Назначение                                                          |
| -------------------------- | ------------------------------------------------------------------- |
| GUID (`XMLСтрока(Ссылка)`) | Постоянный уникальный идентификатор для синхронизации               |
| Код                        | Внутренний код номенклатуры (для контроля)                          |
| Артикул                    | SKU в WooCommerce                                                   |
| Наименование               | Название товара                                                     |
| Единица измерения          | Отображение и логика продаж                                         |
| Пометка удаления           | Чтобы можно было автоматически скрывать или удалять товары на сайте |
| ЭтоГруппа                  | Чтобы не импортировать группы вместо товаров                        |

Такой экспорт будет устойчивым к переименованию товаров и изменению артикулов и позволит безошибочно сопоставлять записи между 1С и WordPress.
